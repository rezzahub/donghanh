## How it works

```tsx
/** @jsxImportSource @donghanh/core */
import { Action, Brief, Display, registerOperation } from "@donghanh/core";

function ListItems({ data }) {
  return (
    <Brief>
      You have {data.items.length} items.
      <Display data={data.items} />
      <Action operation="add-item" label="Add an item" />
    </Brief>
  );
}

export default registerOperation(ListItems, {
  id: "list-items",
  type: "query",
  description: "List all items",
  instruction: "Call this to see all items.",
  input: {},
  responseKey: "listItems",
});
```

## Packages

| Package | Description |
|---------|-------------|
| `@donghanh/core` | JSX runtime, primitives, registry, executor |
| `@donghanh/hono` | Hono routes: `gptRoutes` (GPT Store), `mcpRoutes` (Apps SDK), `chatRoutes` (self-hosted LLM chat) |
| `@donghanh/react` | Headless React hooks for self-hosted chat |
| `@donghanh/widget` | Runtime for Apps SDK iframe widgets |
| `@donghanh/widget-vite` | Vite plugin: bundles widgets + emits manifest |
| `create-donghanh` | CLI scaffolding tool |

## Ship modes

Same operations, three channels:

- **GPT Store** — REST + OpenAPI, runs inside Custom GPTs (`gptRoutes`)
- **Apps SDK** — MCP over HTTP with iframe widgets (`mcpRoutes` + `@donghanh/widget`)
- **Self-hosted** — bring your own LLM, SSE stream to React UI (`chatRoutes` + `@donghanh/react`)

## AI coding skills

Skills for coding agents live in [`skills/`](https://github.com/rezzahub/donghanh/tree/main/skills):

- **`donghanh`** — core concepts, adapters, protocol rules, common pitfalls
- **`donghanh-prompts`** — where to put steering text (description, instruction, start, Brief, widgetDescription)

Install into `~/.claude/skills/` (or your agent's skills dir):

```bash
npx degit rezzahub/donghanh/skills/donghanh ~/.claude/skills/donghanh
npx degit rezzahub/donghanh/skills/donghanh-prompts ~/.claude/skills/donghanh-prompts
```

Then invoke via the Skill tool in Claude Code / Cursor / Copilot CLI when working in a donghanh codebase.
